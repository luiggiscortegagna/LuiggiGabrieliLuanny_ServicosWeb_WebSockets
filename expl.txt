Comentei o código todo, coisa primária pra estudar é client.js e server.js

iniciar codigo com:
npm init -y
npm install express ws
node server.js

A aplicação funciona assim
Cliente web <-> Websocket <-> Servidor Node.js

Cliente web é a página, que tem index.html, style.css e client.js (tudo em public)
Servidor é o server.js

Fluxo:
Usuario abre página -> Digita nome -> client.js abre uma conexão WebSocket -> server.js recebe a conexão -> cliente.js envia o nome escolhido -> server.js valida o nome -> server.js confirma a entrada -> usuário manda mensagem -> server.js recebe e retransmite a mensagem pra todo cliente conectado -> client.js mostra a mensagem na página

A comunicação entre client.js e server.js usa mensagens JSON.

client.js faz os seguintes tipos de envio para o server.js:
Entrar no chat - "entrar" - server.js trata com bloco dados.tipo entrar
Enviar mensagem - "mensagem" - server.js trata com bloco de dados.tipo mensagem
Sair do chat - "sair" - server.js trata com bloco de dados.tipo sair

server.js envia para cliente.js:
Confirmar entrada - "entrada_confirmada" - client.js trata com bloco de dados.tipo entrada_confirmada
Enviar erro - "erro" - client.js trata com bloco dados.tipo erro
Retransmitir mensagem - "mensagem" - client.js trata com bloco dados.tipo mensagem
Aviso automatico - "sistema" - client.js trata com bloco dados.tipo sistema

-- Fluxo completo --
Usuario envia "Olá"

1. Usuario digita no navegador no campo:
<input id="texto" type="text" placeholder="Digite uma mensagem" autocomplete="off">

2. Formulário é enviado, client.js captura o envio:
formulario.addEventListener("submit", ...)

3. Cliente envia pro servidor, cliente usa:
socket.send(JSON.stringify({
  tipo: "mensagem",
  texto: texto
}));

4. Servidor recebe, server.js recebe no evento:
ws.on("message", ...)

5. Servidor identifica o tipo no bloco:
if (dados.tipo === "mensagem")

6. Servidor envia pra todos, server.js chama:
enviarParaTodos({
  tipo: "mensagem",
  nome: ws.nome,
  texto: texto,
  horario: new Date().toLocaleTimeString()
});

7. Todos clientes recebem no navegador o evento:
socket.addEventListener("message", ...)

8. Mensagem aparece na tela, cliente chama:
mostrarMensagem(`[${dados.horario}] ${dados.nome}: ${dados.texto}`, "mensagem");

Resultado na tela:
[20:30:15] Usuario: Olá






index.html
Tem a página em si, e importação do client.js
No bloco <section id="entrada"> o campo "nome" recebe o valor digitado, e o botão "entrar" dispara a criação da conexão WebSocket no client.js

O bloco <section id="chat" hidden> tem o chat que fica escondido, aparece depois que confirmadamente o usuario entrar no chat

O div com id mensagems é onde as mensagens recebidas são inseridas pelo javascript, o formulario com id formulario é usado pra enviar mensagem nova, no final o client.js é carregado (é o arquivo que faz a página se conectar ao servidor websocket)








Server.js
backend da aplicação, gerencia as conexões websocket

Bibliotecas:
express -> servir arquivos html, css e js
http -> criar servidor http base
ws -> criar servidor websocket

o servidor websocket usa o mesmo servidor http da aplicação, na prática http://localhost:3000 serve a página web enquanto a conexão websocket também é aceita pelo mesmo servidor

app.use(express.static("public"));
faz o express servir automaticamente os arquivos da pasta public, entao o navegador pode acessar index.html, client.js, style.css sem precisar criar uma rota manual pra cada coisa

a função enviarParaTodos(dados) envia uma mensagem par atodos os usuários conectados, o parâmetro dados é um objeto javascript (tipo, nome, texto, horario)
como o websocket envia texto, o objeto é convertido pra JSON com JSON.stringify
depois o código percorre todos os clientes conectados, verifica se a conexão está aberta, e então envia a mensagem

função nomeJaExiste(nome) é pra verificar se um nome está em uso consultando o Set.

função removerUsuario(ws) remove um usuário do chat, ele recebe como parâmetro a conexão websocket do usuário, ele remove do set e depois apaga a conexão, por fim envia mensagem de sistema pra todos no chat só pra notificar. isso acontece quando o usuario sai da pagina, perde conexão ou clica em sair

no trecho wss.on("connection", (ws) => {...}); o servidor escuta conexões websocket, sempre que um navegador cria uma conexão com new WebSocket(...) o parâmetro ws representa a conexão daquele cliente, no início começa com nome null que é trocado quando o usuario envia um nome.

o trecho ws.on("message", (data) => {...}); o servidor escuta mensagens vindas daquele cliente, esse evento roda sempre que o cliente enviar algo usando socket.send(...), a primeira coisa feita é converter a mensagem recebida de json pra objeto javascript (código trabalha javascript, websocket transmite json)

ENTRADA NO CHAT:
quando o servidor recebe um {"tipo": "entrar", "nome": "usuario"} ele entra num bloco de dados.tipo "entrar", onde ele faz umas validações: 1. não deixar a mesma conexão websocket entrar duas vezes, 2. remover espaços e transformar o valor recebido em texto, 3. impedir nome vazio, 4. impedir nome repetido
se passar por todas, o nome é salvo na conexão websocket ws.nome e adicionado ao conjunto de nomes em nomesEmUso

depois o servidor confirma para o proprio cliente com ws.send e por fim avisa aos usuarios no chat uma mensagem com enviarParaTodos dizendo que alguem entrou no chat.

BLOQUEIO DE MENSAGENS ANTES DE ENTRAR:
depois do bloco de entrada o servidor verifica se a conexao websocket tem nome, se nao tiver nao pode mandar mensagem. importante pois conexão websocket existe nesse codigo assim que clicar em "entrar", que tambem deve entrar com "nome" mas caso seja burlado de algum jeito para criar a conexao sem entregar um nome, ainda nao pode mandar mensagem.

MENSAGEM DE USUARIO:
quando o cliente envia uma mensagem, ela chega assim {"tipo": "mensagem", "texto": "Olá"}, o servidor entra no bloco dados.tipo "mensagem", primeiro limpa o texto recebido, ignora mensagens vazias, registra no console a mensagem, e envia para todos os usuários conectados com enviarParaTodos. o usuario manda uma mensagem para o servidor, o servidor distribui para todos.

SAIDA:
quando o usuario clica em sair, o cliente envia {"tipo": "sair"}, o servidor entra no bloco sair onde ele libera o nome do usuario e depois fecha a conexao websocket, o servidor tbm tem um ws.on "close" pra remover o nome do usuario caso ele feche a aba ou reinicie ou perca conexao.




client.js
O cliente pode ler os campos da pagina, abrir conexao websocket, enviar dados ao servidor, receber dados do servidor, atualizar a pagina (desocultar/ocultar o chat por ex)

let socket = null pra guardar a conexao websocket atual, quando ele entra no chat ele cria uma conexao new WebSocket(...)
let entrouNoChat = false pra indicar se o servidor ja confirmou a entrada do usuario, vira true quando o servidor envia entrada_confirmada

CAPTURA DE ELEMENTOS HTML:
tem blocos como const entrada = document.getElementById("entrada"); onde cada constante representa um elemento do index.html
ex: mensagensDiv representa <div id="mensagens"></div>
----------------------

A função monstrarMensagem(texto, classe) cria uma div, coloca o texto dentro e adiciona essa div na área de mensagens. o trecho div.textContent = texto define o conteúdo exibido, enqt o div.classList.add(classe); aplica uma classe CSS (sistema, mensagem, erro).

Função limparTela() só volta a pagina pro estado inicial

Botão entrar tem um evento onde ao ser clicado o nome digitado é lido, impedido o vazio, impedido conexão se o usuário já estiver no chat, em seguida trava o botão e o campo pra evitar vários cliques enquanto a conexão está sendo criada

CRIAÇÃO DA CONEXÃO WEBSOCKET NO CLIENTE:
socket = new WebSocket(`ws://${window.location.host}`);
isso abre uma conexao websocket com o mesmo host da página (http://localhost:3000 -> ws://localhost:3000)

Depois que a conexão abre, o evento socket.addEventListener "open" roda, onde o cliente envia ao servidor a intenção de entrar no chat, enviando o tipo "entrar" e seu nome. {"tipo": "entrar", "nome": "usuario"}
então não bastou abrir conexão, cliente ainda tem que enviar o nome pro servidor validar

EVENTO MESSAGE NO CLIENTE:
cliente recebe mensagens do servidor no bloco socket.addEventListener "message", onde converte toda mensagem como json. Depois, cliente verifica o campo tipo.

ENTRADA CONFIRMADA:
quando o servidor aceita a entrada, ele envia {"tipo": "entrada_confirmada", "nome": "usuario"}, o cliente trata disso em if (dados.tipo === "entrada_confirmada"){...} onde ele então marca que o usuário entrou no chat entrouNoChat = true; e esconde a tela de entrada e mostra a tela do chat.

MENSAGENS:
Quando servidor envia erro, o cliente fecha a conexão socket.
Quando o servidor envia mensagem de sistema, o cliente mostra na página no chat.
Quando servidor envia mensagem de usuário, o cliente mostra na página no chat também, só com outra formatação e dados.
Quando a conexão fecha, o bloco socket.addEventListener "close" roda, limpando a tela.
Se houver erro na conexão, o bloco socket.addEventListener "error" roda, mostrando alerta e voltando pro estado inicial
o envio de mensagens ocorre no bloco formulario.addEventListener "submit", primeiro é impedido que a página recarregue, depois verifica se o usuário realmente tá conectado, depois pega o texto digitado, se tudo estiver certo ele envia ao servidor a mensagem.

SAIDA:
quando o usuario clica em sair o cliente envia "sair" ao servidor, que entao remove o usuario e fecha a conexao.
